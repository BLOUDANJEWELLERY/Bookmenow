Calendar Stacking Issue Solution

The Problem

Issue Description

When navigating between months in the calendar component, especially when pressing next, then previous, then next again, the calendar starts to pile up instead of replacing the previous view.

This happens because the animation cleanup doesn't complete properly when quickly navigating between months, causing multiple calendar elements to remain in the DOM.

Problematic Behavior: 
// When quickly navigating:
prevBtn.click();
nextBtn.click();
prevBtn.click();

// Result: Multiple calendar elements stack up in the DOM
// because previous transitions haven't completed

Problem Demo

The Solution

Fix Approach

We'll implement two key improvements:

  1. Animation Lock: Prevent new transitions while one is in progress
  2. Cleanup Mechanism: Properly remove old calendars after animation completes
Solution Code: 
// 1. Add animation lock
let isAnimating = false;

// 2. Update navigation handlers
prevBtn.onclick = () => {
  if (isAnimating) return;
  // ... update month ...
  renderCalendar(year, month, "left");
};

// 3. Add cleanup in animationend
oldCalendar.addEventListener('animationend', (e) => {
  if (e.target === oldCalendar) {
    oldCalendar.remove();
    isAnimating = false; // Release lock
  }
}, { once: true });

// 4. Set animation lock when starting transition
isAnimating = true;

Fixed Solution

Results with Fix

The calendar now properly handles quick navigation between months:

  • Only one calendar element exists in the DOM at any time
  • Smooth animations between month transitions
  • No more stacking of calendar elements
  • Navigation buttons are disabled during transitions
